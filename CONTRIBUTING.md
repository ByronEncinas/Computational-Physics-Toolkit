# How to Contribute 

Anyone can contribute, if you don't know what you wan't to do, search for `good first issue` inn the issues tab.

# Fork the repository

You need to have a personal copy of the code so you can write or modify any file. You can try pushing the fork button, or clone this repository into your local machine.

# Fork guide

If you fork this code, we highly encourage you to focus on the specific task, and avoid merge conflicts in the future. Assuming you are not familiar with git/github, try keeping your modification simple and well localized, not complex and scattered over different files. 

# Clone guide (for main contributors only)

You can clone this repository into your machine. You can create a branch for features under development. Once the feature is properly tested and ready to merge, please make a pull request and another contributor will green light your PR.

---

## General Rules

- One pull request per feature or bug fix — do not bundle unrelated changes
- Do not commit directly to `main`
- Do not modify files unrelated to your change
- Always sync with `upstream main` before starting new work
- If you are unsure about something, open an Issue and ask before writing code
